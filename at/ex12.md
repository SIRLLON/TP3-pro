# Exercicio 12 - Integraçao Final: Serviço HTTP Consolidado

Aqui a gente consolidou todo o aprendizado criando um servidor HTTP que roda sobre TLS de forma segura (HTTPS) no endereço local, respondendo requisiçoes de rotas como `/` e `/health` usando conexoes concorrentes.

## Saida do Servidor no VirtualBox

```
servidor integracao rodando
integracao req: / status: 200 OK
integracao req: /health status: 200 OK
```

## Resposta para a rota / (Curl com TLS Inseguro)

```
HTTP/1.1 200 OK
Content-Type: text/plain
Content-Length: 15
Connection: close

integracao home
```

## Resposta para a rota /health (Curl com TLS Inseguro)

```
HTTP/1.1 200 OK
Content-Type: text/plain
Content-Length: 13
Connection: close

integracao ok
```

## Justificativa Simples

A gente tomou duas principais decisoes de projeto para estruturar esse servidor:
1. **Usar Threads para concorrência**: Escolhemos isso para que mais de uma pessoa consiga acessar o site ao mesmo tempo. Sem threads, se alguem fizesse uma requisiçao, todo mundo teria que esperar na fila.
2. **Rodar com TLS (HTTPS)**: Decidimos colocar segurança no servidor para proteger as informaçoes que trafegam na rede contra bisbilhoteiros, deixando o canal de comunicaçao criptografado e seguro.
